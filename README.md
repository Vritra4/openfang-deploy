# openfang-deploy

Docker deployment and lifecycle management for [OpenFang](https://github.com/RightNow-AI/openfang).

Builds from the latest stable release, runs as a non-root container, and optionally auto-starts on boot via launchd (macOS) or systemd (Linux).

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/)
- [Bun](https://bun.sh/) (v1.0+)
- (Optional) [Tailscale](https://tailscale.com/) for VPN-bound access

## Quick Start

```bash
git clone --recurse-submodules https://github.com/Vritra4/openfang-deploy.git
cd openfang-deploy
bun install

cp .env.example .env && chmod 600 .env
# Edit .env — set LLM keys, etc.

./build   # Build Docker images from latest stable tag
./up      # Start the container
```

## Commands

```
./up [command]
```

| Command | Description |
|---------|-------------|
| `up` | Start container (default) |
| `down` | Stop container |
| `restart` | Stop and start container |
| `logs` | Tail container logs |
| `status` | Show container status |
| `install` | Register auto-start service |
| `uninstall` | Remove auto-start service |
| `service-status` | Show auto-start service status |
| `service-logs` | Tail service startup logs |
| `shutdown` | Stop container + remove auto-start |
| `cleanup` | shutdown + remove Docker images + service/ |
| `cleanup --all` | Also delete `data/` (with confirmation) |

## Configuration

### `.env`

Secrets and environment variables. Never committed to git.

- `OPENFANG_API_KEY` — auto-generated by `build` if empty
- `BIND_IP` — override network binding (see below)
- LLM provider keys, channel integrations, etc.

### `data/config.toml`

OpenFang runtime config. Auto-generated by `build` with sensible defaults. The API key is injected at container startup from `.env`, not stored in this file.

## Network Binding

The container's API port (4200) binds to an IP determined by priority:

1. `BIND_IP` in `.env` — explicit override (any VPN, static IP, `0.0.0.0`)
2. Tailscale — auto-detected if installed and connected
3. `127.0.0.1` — localhost fallback

## Auto-Start Service

Register OpenFang to start automatically on boot/login:

```bash
./up install    # macOS: launchd, Linux: systemd --user
./up uninstall  # Remove
```

The service waits for Docker (and Tailscale, if no `BIND_IP` is set) before starting the container.

On Linux, for boot-time start without login:

```bash
loginctl enable-linger $USER
```

## Project Structure

```
openfang-deploy/
├── openfang/          # Source (git submodule)
├── data/              # Persistent data (gitignored)
├── service/           # Auto-start scripts (generated)
├── build           # Build Docker images
├── up              # Container lifecycle management
├── Dockerfile         # Wrapper image (non-root user)
├── entrypoint.sh      # Runtime config injection
├── docker-compose.yml # Container orchestration
└── .env.example       # Environment template
```

## Updating

```bash
./build   # Pulls latest stable tag + rebuilds
./up restart
```
